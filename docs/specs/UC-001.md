# UC-001: Spiellinie generieren (KI-gestützt)

## Kurzbeschreibung
Ein Benutzer erstellt eine neue, kreative Spiellinie für das "3LandSpiel", indem er grundlegende Parameter definiert. Das System stellt durch einen zwischengeschalteten Guardrail-Layer sicher, dass die generierten Inhalte didaktisch wertvoll, ethisch korrekt und thematisch passend sind, bevor sie durch ein Large Language Model (LLM) erzeugt werden.

## Akteure
*   **Benutzer:** Lehrperson, Game Master oder pädagogische Fachkraft.
*   **LLM (System-Akteur):** Die KI-Schnittstelle zur Textgenerierung.

## Vorbedingungen
*   Das System ist betriebsbereit.
*   Die Verbindung zum LLM ist konfiguriert und aktiv.
*   Die Guardrail-Konfigurationen (ethisch, didaktisch) sind hinterlegt.

## Standardablauf (Happy Path)
1.  **Parameter eingeben:** Der Benutzer ruft das Interface zur Generierung auf und gibt Zielgruppe (z.B. "Primarschule") und ein grobes Thema (z.B. "Umweltschutz am Rhein") ein.
2.  **Anfrage absenden:** Der Benutzer bestätigt die Eingabe.
3.  **Guardrail-Prüfung:** Das Backend empfängt den Prompt und ergänzt ihn durch den Guardrail-Layer mit spezifischen Anweisungen (Kontextfilter, Inhaltsbeschränkungen, didaktische Richtlinien).
4.  **KI-Generierung:** Der gesicherte und angereicherte Prompt wird an das LLM gesendet.
5.  **Verarbeitung:** Das LLM generiert den Inhalt basierend auf den Vorgaben.
6.  **Ergebnis anzeigen:** Das System zeigt die generierte Spiellinie (z.B. Story-Plot, Aufgabenstellungen, Rollen) im UI an.

## Alternative Abläufe & Fehlerfälle
*   **A-1: Guardrail-Block:** Wenn die Benutzereingabe gegen grundlegende Sicherheitsrichtlinien verstößt (z.B. unangemessene Sprache), wird die Anfrage abgebrochen und der Benutzer erhält einen Hinweis.
*   **A-2: LLM-Timeout:** Falls das LLM nicht antwortet, informiert das System den Benutzer und schlägt einen erneuten Versuch vor.

## Akzeptanzkriterien
*   [ ] Die UI bietet Eingabefelder für Zielgruppe und Thema.
*   [ ] Der Guardrail-Layer fügt nachweislich vordefinierte System-Prompts (ethisch/didaktisch) hinzu.
*   [ ] Das System sendet keine Rohdaten des Benutzers direkt an das LLM ohne Guardrail-Intervention.
*   [ ] Die generierte Spiellinie wird innerhalb einer angemessenen Zeit (< 15s) im UI angezeigt.
*   [ ] Der Inhalt entspricht der gewählten Zielgruppe (didaktische Konsistenz).
